#### 影音方案

最终方案：**网易爆米花 + 115网盘**。彻底解耦家里云的依赖，观影体验丝滑。

Tips：爆米花扫盘时，别直接扫根目录——只扫新入库的最底层目录，能省掉大量全盘扫描的开销。

#### 相册备份方案

这条链路稍微绕一点，但安全性拉满：

```
手机 → MTPhoto 自动备份 → NAS 本地目录(储存点A)
                              ↓
                        OpenList 本地挂载
                              ↓
                        OpenList 网盘挂载(储存点B) → 加密(储存点C)
                              ↓
                        Taosync 定时同步 A → C
```

最终效果：网盘上存的全是加密文件，但通过 WebDAV 挂载储存点 C 到 NAS 后，可以直接免解密浏览照片。细节参考 [OpenList 文档 - 加密](https://doc.oplist.org/guide/drivers/crypt)。

## AI 订阅

聊聊当前的 AI 基础设施配置：

**Agent 基座** — 雷大善人的 100T 计划里的 `Mimo-V2.5-Pro`，还有十来天到期，根本烧不完。到期后切 DeepSeek 官方 API：日常用 `DeepSeek-V4-Flash`，交易 Agent 和 SchemaBio 业务用 `DeepSeek-V4-Pro`。DeepSeek 便宜大碗，而且我给梁圣付费的意愿确实很高。

**日常任务** — 马大善人的 WorkBuddy，主力模型 `GLM-5.2`。让它自己开发了一个对接服务器的 SKILL，接到微信渠道。现在已经能做到通过微信下发日常简单工作任务。

**Coding 场景** — Codex，接 Any 大善人的端点。对生信场景来说，每天 $25 额度的 `GPT-5.5`，用不完，根本用不完。



那为什么不直接上 Token Plan 或者 Coding Plan？踩过坑，说说我的体感：

1. **上新太慢** — 模型迭代速度是按周算的，别人已经吃上了最新的顶级开源模型，自己的 Coding Plan 还在排队等更新，心态会崩
2. **用不完** — 实测连 Lite 套餐都烧不满。当然这跟我日常任务和 Coding 场景不走 Plan 有关，但确实性价比存疑
3. **天花板问题** — 即便 Plan 能提供最好的开源模型，和闭源的 GPT / Claude 在复杂任务上仍有差距，该切还得切

所以最终结论：**按量付费 + DeepSeek**，反而成了我的最优解。灵活、便宜、没有沉没成本。
